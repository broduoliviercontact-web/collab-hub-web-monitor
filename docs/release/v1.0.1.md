## Procédure d'installation

### Web (développement local)

```bash
npm install
cp .env.example .env       # vérifier VITE_COLLAB_HUB_NAMESPACE=hub
npm run dev                # http://localhost:5173
```

### Max

1. Télécharger et dézipper `CollabHub-Web-Monitor-Max-v1.0.1.zip`.
2. Installer le client Collab-Hub Max Client
   (https://github.com/Collab-Hub-io/Collab-Hub-Max-Client) dans le Search Path
   de Max (**Options → File Preferences**, cocher Subfolders).
3. Ouvrir `CollabHub_Web_Text_Sender.maxpat` dans Max.
4. Connecter le CH-Client à `https://server.collab-hub.io`, attendre la version
   0.3.4.
5. Remplir les cinq champs (valeurs entre guillemets), cliquer **ENVOYER LES 5
   CHAMPS**.
6. Ouvrir le site web → les blocs se remplissent, statut « Connecté — Max
   actif ».

Voir `README-Max.md` dans le ZIP.

## Limites connues

- La 1re publication d'un header l'enregistre seulement (le patch envoie chaque
  champ deux fois, register + deliver).
- Persistance locale par navigateur uniquement (pas d'historique, pas de
  multi-postes).
- `maxLastSeenAt` non restauré au rechargement → Max « silencieux » jusqu'au
  prochain heartbeat (≤ 10 s).
- Serveur public v0.3.4 vieillissant ; auth invitée absente (mode `anonymous`
  par défaut, aucune requête d'auth).
- Diagnostic `?debug=1` accessible sans login (à protéger en production).
- Aucune clause supplémentaire ou restriction ajoutée à la GPL-3.0-only.
